## STEP 1. 오케스트레이터 실행

아래 오케스트레이터를 읽고 순서대로 진행하세요:
`~/.claude/commands/ld-deal-plugin/skills/오케스트레이터.md`

설정 파일: `~/.claude/commands/ld-deal-plugin/config/settings.json`

스크립트 경로: `~/.claude/commands/ld-deal-plugin/scripts/`

실행 흐름:

1. **Phase 0:** `python scripts/check_env.py --settings config/settings.json` — DB 자동 다운로드(urllib) + MCP 확인
2. **Phase 1:** 세일즈맵에서 `settings.target_ld.name` LD의 **액티브 딜만** 조회 (단계 6종 + 성사 가능성 ≠ LOST) + memo 테이블 join
3. **Phase 2:** 캘린더·슬랙·지메일·드라이브 견적서·기획문서 **병렬 수집** → 스킬별 중간파일 저장
   - 슬랙 채널은 `settings.data_sources.slack_channels` (2팀 견적제안·제안노트 등)
4. **Phase 2.5:** LLM이 거래 의지 카테고리(C1·C2·C3) + 마감 추출 + `성사 가능성` grounding 비교
5. **Phase 3:**
   - `python scripts/merge_deals.py --mode skills` (또는 db 모드 — 검증 시)
   - `python scripts/calculate_score.py deals.json --settings config/settings.json` (7기준, **만점 170**)
   - `python scripts/verify_scores.py` (**6항목 검증**)
6. **Phase 3.5:** LLM이 reason + next_action 일괄 생성 (티어별 톤: T1 3줄·T2 2줄·T3 1줄)
7. **Phase 4a:** `archive/YYYYMMDD.json` 스냅샷 저장
8. **Phase 4b:** `python scripts/detect_changes.py --week archive --db ~/salesmap/salesmap_latest.db` (Won 전환 구분)
9. **Phase 4c:** `python scripts/generate_md.py` → `outputs/summary_report_YYYYMMDD.md`
