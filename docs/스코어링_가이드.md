# 딜 요약 플러그인 — 스코어링 가이드

> "왜 이 딜이 T1인가?"를 LD가 이해하고 필요 시 기준을 조정할 수 있도록 만든 참고 문서.

---

## 1. 스코어링 개요

본 플러그인은 **SQL 단계 딜의 우선순위를 정량화**합니다.
핵심 원칙: **"검증은 코드, 판단은 사람"**
- 점수 계산 = Python 스크립트 (결정론적, 재현 가능)
- 근거 생성 = LLM (맥락 해석)
- 최종 의사결정 = LD 본인

**만점 155점** (가점 최대) / 감점 최대 -35점 / 실제 범위 -35 ~ 155점

## 2. 7개 기준 상세

#### 기준 1. 딜 금액 (최대 +30)
| 레벨 | 조건 | 점수 |
|---|---|---|
| 상 | 3억 이상 | +30 |
| 중상 | 1억 이상 | +20 |
| 중 | 5천만 이상 | +15 |
| 하 | 3천만 이상 | +10 |
| 최하 | 그 외 | +5 |

> 딜 금액은 매출 기여 직접 반영. `settings.scoring.amount_threshold_high/mid` 로 조정 가능.

#### 기준 2. 파이프라인 단계 (최대 +25)
| 단계 | 점수 | 의미 |
|---|---|---|
| 최종 f-up | **+25** | 성사 직전 |
| 2차 f-up | **+18** | 후속 진행 중 |
| Proposal 송부 | +15 | 제안서 송부 완료 |
| Proposal 준비 | +10 | 준비 단계 |
| 1차 f-up | +7 | 초기 접촉 |

> 2026-04-20 재조정: 2차 f-up vs Proposal 송부 변별력 확보, 1차 f-up 하향.

#### 기준 3. 마감 임박도 (최대 +30, 감점 -10)
| 조건 | 점수 |
|---|---|
| D-7 이내 | +30 |
| D-14 이내 | +25 |
| D-30 이내 | +15 |
| D-60 이내 | +10 |
| D-60 초과 | +5 |
| **마감 지남** | **-10** |

> 세일즈맵 `수주 예정일` 기준. 미입력 시 +5 (기본값).

#### 기준 4. 고객 가치 (최대 +25)
**기고객 이력 (최대 +15):**
| 거래 건수 | 점수 |
|---|---|
| 20건+ | +15 |
| 10건+ | +12 |
| 5건+ | +8 |
| 1건+ | +5 |

**전략 고객 가산 (최대 +10):**
- `settings.scoring.strategic_keywords` 매칭 시 +10 (삼성·현대·KT·SK·LG 등)
- 또는 누적 10억+ 거래 시 +5

> 합계 25점 상한.

### 보조 3개 (같은 점수대 딜 세부 순위 조정)

#### 기준 5. 소통 활성도 (최대 +20, 감점 -20)
**라스트 터치 경과일:**
| 조건 | 점수 |
|---|---|
| 3일 이내 | +15 |
| 7일 이내 | +10 |
| 14일 이내 | +5 |
| 30일 이내 | 0 |
| 60일 이내 | -10 |
| 60일+ | -20 |

**고객 반응 가산:**
- 고객 회신 있음: +5
- 30일+ 무응답: -5

#### 기준 6. 이번 주 액션 (최대 +15)
| 조건 | 점수 |
|---|---|
| 이번 주 미팅 있음 | +15 |
| 이번 주 액션 예정 | +10 |
| 없음 | 0 |

#### 기준 7. 딜 출발점 + 성사 가능성 (최대 +20)

**딜 출발점 (base):**
| 유입 경로 | 점수 |
|---|---|
| RFP | +20 |
| 소개/추천 | +20 |
| 기고객 재의뢰 | +20 |
| 인바운드 | +10 |
| 아웃바운드 | +5 |
| 다수 업체 비교 | +5 |
| 가격 비교만 | 0 |

**성사 가능성 보정 (Option B, LD 직접 판단):**

- 세일즈맵 `성사 가능성 = 낮음` → **-5 down** (하한 0)

> LD가 세일즈맵에 직접 입력한 값으로 LLM 판단을 보정. 가장 현장 가까운 시그널.

## 3. 티어 분류

| 티어 | 점수 기준 | 명칭 | 행동 가이드 |
|---|---|---|---|
| 🔴 T1 | 80점+ | 집중 | 오늘 즉시 대응 |
| 🟠 T2 | 50~79점 | 관리 | 이번 주 관계 육성 |
| ⚪ T3 | 50점 미만 | 지켜보기 | 주기적 모니터링 |

`settings.scoring.t1_threshold`·`t2_threshold`에서 조정 가능.

### ⭐T1 후보 뱃지 (하이브리드 가드)

**조건:** 점수 70~79점 AND 마감 D-30 이내 AND ([핵심기업] OR [재거래])

- **강제 승급 아님** — T2에 그대로 유지
- 뱃지로 "상황 따라 T1 수준 대응 검토" 시각 표시
- 경계선 딜의 LLM 판단 편차 완충

`settings.scoring.t1_candidate_*` 에서 조건 조정 가능.

---

## 4. 뱃지 체계

카드에 텍스트 `[...]` 형식으로 표시:

| 뱃지 | 조건 | 의미 |
|---|---|---|
| `[핵심기업]` | `is_strategic_customer: true` | 대형 고객사 (strategic_keywords 매칭) |
| `[신규딜]` | SQL 전환 7일 이내 | 최근 유입된 딜 |
| `[재거래]` | 과거 성사 딜 1건+ | 기고객 |
| `⭐T1 후보` | 위 가드 조건 충족 | 경계선 딜, 상황 따라 우선 대응 |

## 5. 자주 묻는 질문

### Q1. 왜 어제 T1이었던 딜이 오늘 T2로 내려갔나?
A. LLM이 `deal_origin`이나 소통 상태를 해석할 때 10점 내외 편차 발생 가능. 경계선 딜(80점 부근)은 실행마다 흔들릴 수 있음. `⭐T1 후보` 뱃지는 이 편차 완충용.

### Q2. 점수는 80점인데 `⭐T1 후보`는 왜 안 붙었나?
A. `⭐T1 후보`는 **70~79점 + 조건 충족** 시에만 붙음. 80점 이상은 정식 T1이라 뱃지 불필요.

### Q3. 기준 자체를 바꾸고 싶다
A. `config/settings.json`에서 조정:
- `scoring.t1_threshold`, `t2_threshold` — 티어 커트라인
- `scoring.amount_threshold_high/mid` — 금액 구간
- `scoring.strategic_keywords` — 핵심기업 키워드
- `scoring.t1_candidate_*` — ⭐T1 후보 조건

### Q4. 어떤 기준이 가장 큰 영향을 주나?
A. 배점 기준: 딜 금액(30) == 마감 임박도(30) > 파이프라인/고객가치(25) > 소통/출발점(20) > 이번 주 액션(15)

### Q5. LLM이 틀린 판단을 한 것 같다
A. reason/next_action에 "이건 맞지 않다" 생각되면 PR/이슈로 공유. 다만 점수 자체는 스크립트 결정이라 LLM이 못 바꿈 — 입력 데이터(deal_origin, last_touch_days 등) 문제일 가능성 큼.

### Q6. SQL→Won 전환되어 리포트에서 사라진 딜은?
A. "🔄 최근 변화" 섹션에 "🎉 수주 성공"으로 표시됨. detect_changes.py가 세일즈맵 DB에서 현재 상태 확인 후 구분.
