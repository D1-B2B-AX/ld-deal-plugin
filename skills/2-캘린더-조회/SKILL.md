# 스킬 2: 캘린더 조회

> 스킬 1에서 빠진 **시간 정보를 보충**: 이번 주/다음 주에 잡혀있는 딜 관련 미팅, 팔로업 예정을 가져옴.

## 역할

스킬 1(세일즈맵)의 **보조 스킬**. 세일즈맵에는 없는 아래 정보를 채움:
- 이번 주 딜 관련 미팅/콜 예정 → "오늘 미팅 있으면 이 딜이 급함"
- fu(팔로업) 예정일 → 계획된 액션 존재 시그널

## 인풋

| 파라미터 | 출처 | 설명 |
|---------|------|------|
| `search_keywords` | 스킬 1 아웃풋 | 딜별 고객사명, 딜카드명 토큰 |
| `calendar_id` | `settings.target_ld.calendar_id` | LD 본인 캘린더 ID (다중 LD 지원) |
| `time_range` | 고정 | 오늘 ~ 2주 뒤 |

## 처리 흐름

```
STEP 1: 이번 주 + 다음 주 전체 이벤트 조회 (1회 호출)
  ↓
STEP 2: 딜 매칭 — 이벤트 제목에 고객사명/딜 키워드 포함 여부
  ↓
STEP 3: 매칭 안 되는 이벤트 자동 제외
  ↓
STEP 4: 딜별 이벤트 리스트로 구조화
```

## 매칭 규칙

- **매칭 기준**: 이벤트 제목(summary)에 `search_keywords.organization_name` (고객사명) 포함 여부로 판단
- **범용 키워드 사용 안 함**: "AI", "교육" 같은 흔한 단어로는 매칭하지 않음 (무관한 일정이 잘못 매칭되는 것 방지)
- **동일 고객사 복수 딜**: 양쪽 모두 중복 표시 (잘못 구분하는 것보다 중복이 안전)
- **자동 제외**: 매칭 안 되는 이벤트 (개인 일정, "사무실", "출근", "No Title" 등)
- **예시 매칭**: "한화솔루션 성민님 싱크업" → organization_name "한화솔루션" 매칭 ✅

## 아웃풋

딜별로 매칭된 이벤트 배열:

```json
{
  "deal_id": "019cb7ee-d0bc-7ee0-a2af-6829fe1e5ee8",
  "deal_name": "한화솔루션_AX러닝랩 프로젝트 교육",
  "matched_events": [
    {
      "date": "2026-04-22",
      "time": "09:30-10:00",
      "title": "한화솔루션 성민님 싱크업",
      "attendees": ["{owner.email}", "..."],
      "type": "미팅",
      "days_from_today": 2
    }
  ]
}
```

매칭된 이벤트가 없는 딜은 `"matched_events": []` (빈 배열).

### 필드 의미

| 필드 | 용도 |
|------|------|
| `matched_events` | 스킬 5 (우선순위 판단) — "이번 주 미팅 있음 = 급한 딜" 시그널 |
| `type` | "미팅" / "교육" / "fu" / "기타" — 이벤트 제목에서 키워드로 자동 분류 |
| `days_from_today` | 스킬 5 — 임박도 시그널 (0=오늘, 1=내일 ...) |

## 이벤트 타입 자동 분류 규칙

| 제목 키워드 | 분류 |
|-----------|------|
| 싱크업, 미팅, 회의, 콜, 방문 | **미팅** |
| 교육, 강의, 워크숍, 세션 | **교육** |
| fu, f-up, 팔로업, 메일 보내기 | **fu** |
| 위 해당 없음 | **기타** |

## MCP 호출

```
도구: mcp__workspace-mcp__get_events
파라미터:
  user_google_email: settings.target_ld.email
  calendar_id: settings.target_ld.calendar_id (또는 "primary")
  time_min: (오늘 00:00 KST)
  time_max: (14일 후 00:00 KST)
  max_results: 50
  detailed: true
```

## 알려진 제약·주의사항

| 항목 | 내용 |
|------|------|
| 비공개 일정 | "No Title"로 표시 — 자동 제외 |
| 매칭 누락 가능 | 이벤트 제목에 고객사명이 약어/영문으로 적혀있으면 못 잡음 (예: "HDS 미팅" → "현대건설" 매칭 안 됨) |
| 다른 사람 캘린더 | LD 본인 캘린더만 조회 (타인 구독 불필요) |
| 조회 범위 | 2주 고정 (이번 주 + 다음 주) |

## LD 체크 포인트

1. 캘린더 이벤트 제목에 **고객사명을 일관되게 포함**하는 네이밍 습관 필요 → 매칭 정확도에 직접 영향
2. 약어·영문명 사용 시 매칭 누락 가능 — 가급적 한글 정식명 사용 권장

## 기존 플러그인과의 차이

- Won 딜(교육 운영) 관련 캘린더 매칭 로직 제거 — SQL 단계 딜만 대상
- 다차수 교육 회차 날짜 추출 로직 제거 — 운영 무관
- calendar_id 하드코딩 → settings 외부화
